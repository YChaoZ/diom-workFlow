## 🚀 5分钟快速测试

### 第一步：打开浏览器

访问: **http://localhost:3000**

应该看到精美的登录页面 🎨

---

### 第二步：登录系统

使用默认账号：

```
用户名: admin
密码: 123456
```

点击"登录"按钮

✅ **预期结果**: 登录成功，跳转到首页，显示统计数据

---

### 第三步：发起请假流程

1. 点击左侧菜单 **"工作流管理" → "流程定义"**
2. 看到"请假审批流程"，点击 **"发起流程"**
3. 填写表单：
   ```
   请假天数: 3
   请假原因: 测试流程，因个人事务需要请假
   ```
4. 点击 **"提交"**

✅ **预期结果**: 
- 提示"流程启动成功"
- 自动跳转到"流程实例"页面
- 列表中显示刚才创建的流程实例

---

### 第四步：切换用户处理任务

#### 4.1 退出当前用户

1. 点击右上角头像
2. 选择 **"退出登录"**
3. 确认退出

#### 4.2 使用经理账号登录

```
用户名: user_1
密码: 123456
```

#### 4.3 查看待办任务

1. 登录后，首页应该显示有待办任务
2. 点击 **"工作流管理" → "我的任务"**
3. 看到"经理审批"任务

✅ **预期结果**: 任务列表显示待处理的请假审批任务

---

### 第五步：处理任务

1. 点击任务的 **"处理"** 按钮
2. 查看任务详情，包括：
   - 申请人: admin
   - 请假天数: 3天
   - 请假原因: 测试流程，因个人事务需要请假
3. 填写审批意见：
   ```
   审批结果: 选择"同意"
   审批意见: 同意请假申请，注意按时返回
   ```
4. 点击 **"提交"**

✅ **预期结果**: 
- 提示"任务提交成功"
- 返回任务列表
- 该任务从列表中消失（已完成）

---

### 第六步：查看流程历史

1. 切换回 admin 账号（重复第四步 4.1 和登录步骤）
2. 点击 **"工作流管理" → "流程实例"**
3. 找到刚才的流程实例，点击 **"查看历史"**

✅ **预期结果**: 
- 显示流程执行的时间线
- 包含：开始事件 → 经理审批 → 结束事件
- 显示每个节点的处理人和时间

---

### Q1: 前端无法访问？

**A**: 检查是否已启动开发服务器

```bash

```

### Q2: 登录失败？

**A**: 检查后端服务是否正常运行

```bash
# 检查各服务健康状态
curl http://localhost:8081/actuator/health
curl http://localhost:8082/actuator/health
curl http://localhost:8083/actuator/health
curl http://localhost:8080/actuator/health
```

### Q3: 页面显示异常？

**A**: 
1. 清除浏览器缓存
2. 使用 Chrome 浏览器（推荐）
3. 打开浏览器控制台查看错误信息
